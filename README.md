# ВАТТ.МАРКЕТ — сайт-каталог электротоваров и освещения

Полноценный сайт на Next.js: каталог товаров (импорт из Excel), корзина без регистрации
и оплаты, оформление заказа через форму "Имя + контакт", уведомление о заказе в Telegram,
защищённая паролем админка для загрузки Excel-файлов.

## Что внутри

- **Каталог** — карточки товаров, фильтр по категориям, поиск
- **Корзина** — добавление/удаление товаров, без регистрации
- **Оформление заказа** — форма "Имя + телефон/Telegram", при отправке заказ:
  1. сохраняется в базу данных (таблица `orders`)
  2. отправляется вам в Telegram через бота
- **Админка** (`/admin`) — вход по логину/паролю, загрузка Excel-файла с товарами,
  список товаров с удалением, список заказов
- **База данных** — SQLite, файл `data/shop.db` (создаётся автоматически)

## 1. Установка (локально или на сервере)

Понадобится **Node.js версии 18.18 или новее**. Проверить версию: `node -v`.

```bash
cd electro-shop
npm install
cp .env.example .env
```

Откройте `.env` и заполните:
- `ADMIN_LOGIN`, `ADMIN_PASSWORD` — логин и пароль для входа в админку
- `ADMIN_JWT_SECRET` — любая длинная случайная строка (для безопасности сессии)
- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` — см. раздел 3 ниже

Инициализируйте базу данных:

```bash
npm run init-db
```

## 2. Запуск

**Для разработки/проверки на своём компьютере:**
```bash
npm run dev
```
Сайт откроется на http://localhost:3000, админка на http://localhost:3000/admin

**Для продакшена на сервере (VPS):**
```bash
npm run build
npm start
```
По умолчанию сайт поднимется на порту 3000. Чтобы открыть его на 80/443 порту с доменом
и HTTPS, поставьте перед ним Nginx как reverse proxy (см. раздел 5) и используйте
**pm2**, чтобы процесс не падал и перезапускался сам:

```bash
npm install -g pm2
pm2 start npm --name "electro-shop" -- start
pm2 save
pm2 startup
```

## 3. Настройка Telegram-бота (приём заказов)

Никакого постоянно работающего бота писать не нужно — сайт просто отправляет сообщение
через Telegram Bot API при каждом заказе. Это значит, что бот будет работать на любом
обычном хостинге, где крутится сам сайт.

1. В Telegram напишите **@BotFather**, отправьте команду `/newbot`, придумайте имя — получите
   **токен** вида `123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Вставьте его в `.env` как
   `TELEGRAM_BOT_TOKEN`.
2. Напишите своему новому боту любое сообщение (например "привет") — это нужно, чтобы бот
   "увидел" ваш чат.
3. Откройте в браузере (подставив свой токен):
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
4. В ответе найдите `"chat":{"id": 123456789, ...}` — это число и есть ваш `TELEGRAM_CHAT_ID`.
   Вставьте его в `.env`.
5. Перезапустите сайт (`npm run dev` или `pm2 restart electro-shop`).

После этого при каждом заказе вам в личку от бота будет приходить сообщение с именем,
контактом покупателя и списком товаров с ценами.

> Если хотите получать заказы в группу, а не в личку — добавьте бота в группу
> администратором и узнайте `chat_id` группы тем же способом через `getUpdates`
> (он будет отрицательным числом, например `-1001234567890`).

## 4. Загрузка товаров из Excel

1. Зайдите на `/admin`, войдите по логину/паролю из `.env`.
2. На вкладке "Загрузка Excel" выберите файл `.xlsx` и нажмите "Загрузить файл".

**Структура файла** — пример лежит в `data/template_tovary.xlsx`, скопируйте и
заполните его. Столбцы (порядок не важен, можно на русском или английском):

| Столбец | Обязателен | Описание |
|---|---|---|
| Название | Да | Название товара |
| Категория | Нет | Если не указана — попадёт в "Без категории" |
| Цена | Да | Число, можно с "₽" или пробелами — распознается автоматически |
| Артикул | Нет | Используется для обновления товара при повторной загрузке |
| Описание | Нет | Показывается на странице товара |
| Наличие | Нет | "Да"/"Нет" — если не указано, считается "в наличии" |
| Фото (ссылка) | Нет | Прямая ссылка на изображение в интернете |

**Режимы загрузки:**
- *Заменить весь каталог* — старые товары удаляются, загружаются новые из файла
- *Обновить по артикулу* — товары с существующим артикулом обновляются, новые добавляются,
  остальные не трогаются (удобно для регулярного обновления цен/наличия)

### Про фото

Сейчас фото берутся по прямой ссылке (URL) из столбца "Фото". Чтобы не хранить фото на
вашем компьютере, удобно один раз загрузить все фото в любое облачное хранилище с прямыми
ссылками (например, Яндекс.Диск с публичной ссылкой, Imgur, или раздел "Медиа" вашего
хостинга) и проставить ссылки в Excel. Если фото нет — карточка покажет иконку-заглушку,
сайт от этого не сломается.

Когда решите перейти на загрузку файлов фото напрямую с компьютера — дайте знать, я добавлю
загрузку файлов в админку (просто более трудоёмкая функция, отложили на потом по вашей
просьбе).

## 5. Деплой на Railway (самый простой способ)

1. Залейте проект в репозиторий на GitHub (`git init`, `git add .`, `git commit`, `git push`).
2. На [railway.app](https://railway.app) — **New Project → Deploy from GitHub repo**, выберите репозиторий.
3. Вкладка **Variables** — добавьте: `ADMIN_LOGIN`, `ADMIN_PASSWORD`, `ADMIN_JWT_SECRET`,
   `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `NODE_ENV=production`.
4. Вкладка **Settings → Volumes** — добавьте volume с mount path `/app/data`, иначе каталог
   товаров и заказы будут обнуляться при каждом передеплое.
5. Railway сам соберёт и запустит проект (используется `railway.json` из репозитория).
   После деплоя во вкладке **Settings → Networking** нажмите **Generate Domain** —
   получите публичный адрес вида `electro-shop-production.up.railway.app`.

## 6. Деплой на VPS с доменом (пример для Ubuntu + Nginx)

```bash
# на сервере
sudo apt update && sudo apt install -y nginx
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
sudo apt install -y nodejs

# клонируйте/скопируйте проект на сервер, затем:
cd electro-shop
npm install
npm run build
npm install -g pm2
pm2 start npm --name electro-shop -- start
pm2 save && pm2 startup
```

Настройте Nginx (`/etc/nginx/sites-available/electro-shop`):

```nginx
server {
    listen 80;
    server_name ваш-домен.ру;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/electro-shop /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
# Бесплатный HTTPS-сертификат:
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d ваш-домен.ру
```

Альтернативы VPS без ручной настройки сервера: **Railway**, **Render** — там можно
задеплоить проект из Git-репозитория за пару кликов, переменные окружения задаются
в панели управления, бесплатного тарифа обычно достаточно для старта.

## 8. Частые вопросы

**Можно ли изменить цвета/шрифты/логотип?**
Да, всё в `app/globals.css` (цветовые переменные в начале файла) и в `components/Header.js`.

**Как изменить пароль админки после установки?**
Просто измените `ADMIN_PASSWORD` в `.env` и перезапустите сайт.

**Заказ не приходит в Telegram, что проверить?**
1. Правильно ли указан `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в `.env`
2. Писали ли вы боту хотя бы одно сообщение (без этого бот не может писать вам первым)
3. Перезапущен ли сайт после изменения `.env`
4. Логи сервера (`pm2 logs electro-shop`) покажут ошибку, если она есть — заказ при этом
   всё равно сохранится в базу данных и будет виден на вкладке "Заказы" в админке.
